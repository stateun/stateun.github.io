## 개요

본 포스팅에서는 **MIMIC-IV** 데이터를 활용하여 **급성 신손상(AKI)을 예측**하기 위한 **Deep Semi-Supervised Anomaly Detection(DeepSAD)** 기반 모델과, 해당 모델에 **공정성(Fairness) 로스**를 추가하여 새로운 목적함수를 구성한 과정을 정리합니다.  

---

## 연구 동기

- **급성 신손상(AKI)** 발생 여부는 0(정상) / 1(이상)로 라벨링 가능
- 데이터 EDA 결과, 약 20%의 비율로 AKI 발생이 확인됨
- 이를 **이상치(Anomaly)** 로 간주하고, **DeepSAD**를 통한 이상 탐지를 시도
- 모델의 예측에서 특정 집단(보호 대상 집단)이 불리함을 겪지 않도록 **공정성(Fairness) 측면**을 함께 고려

---

## DeepSAD 란?

[**Deep Semi-Supervised Anomaly Detection**](https://arxiv.org/abs/2003.12753)은 **Support Vector Data Description (SVDD)** 접근법을 딥 뉴럴 네트워크와 결합한 이상 탐지 기법입니다.  
- 신경망으로 임베딩을 수행한 뒤, 정상 데이터가 임베딩 공간에서 특정 중심으로 모이도록 훈련
- 이상치로 판단되는 샘플(여기서는 AKI 발생)은 중심으로부터 멀어지도록 유도

### DeepSAD 표준 목적함수

일반적으로 DeepSAD 모델은 다음과 같은 형태의 목적함수를 최소화합니다. (아래 식은 개념적 예시입니다.)

\[
\mathcal{L}_{\text{DeepSAD}} = \sum_{i=1}^N \left\| f_\theta(x_i) - c \right\|^2 + \lambda \left\|\theta\right\|^2
\]

- \( f_\theta(\cdot) \): 입력 \( x_i \)를 임베딩 공간으로 매핑하는 딥 뉴럴 네트워크
- \( c \): 정상 데이터들이 모여야 할 임베딩 공간 상의 중심(center)
- \( \theta \): 네트워크 파라미터
- \( \lambda \): 정규화 계수

## Fairness Loss 추가

공정성(Fairness)을 고려하는 지표로는 **DP( Demographic Parity )**, **EO( Equalized Odds )** 등이 대표적입니다.  
- **DP(Demographic Parity)**: 보호대상 집단과 비보호대상 집단 간에 긍정(Positive)으로 예측되는 비율이 유사해야 함  
- **EO(Equalized Odds)**: 실제 양성(Positive)인 경우와 실제 음성(Negative)인 경우 각각에서 보호대상 집단과 비보호대상 집단의 예측 양성 비율이 유사해야 함  

이들 공정성 지표를 만족시키기 위해서는, 지표별 손실(\(\mathcal{L}_{DP}\), \(\mathcal{L}_{EO}\))을 모델 학습 과정에서 최소화하도록 반영할 수 있습니다.

예를 들어, Demographic Parity를 위한 손실 항이 \(\mathcal{L}_{DP}\), Equalized Odds를 위한 손실 항이 \(\mathcal{L}_{EO}\)라 하면, 최종적인 목적함수는 아래처럼 정의할 수 있습니다.

\[
\mathcal{L}_{\text{DeepSAD+Fair}} 
= \mathcal{L}_{\text{DeepSAD}} + \alpha \times \mathcal{L}_{\text{Fair}}
\]

- \(\mathcal{L}_{\text{Fair}}\)는 \(\mathcal{L}_{DP}\) 또는 \(\mathcal{L}_{EO}\)가 될 수 있음  
- \(\alpha\)는 Fairness Loss와 DeepSAD Loss의 균형을 조절하는 하이퍼파라미터

## MIMIC-IV 데이터 소개

**MIMIC-IV**는 환자의 개인 식별 정보가 제거된 EHR(전자 건강 기록) 공개 데이터셋으로, ICU 및 병원 전반의 입원 기록 등이 포함되어 있습니다.

- **환자 수**: 약 364,627명
- **입원 건수**: 546,028번의 입원 기록과 94,458번의 ICU 입원 기록
- **전체 이벤트**: 43,000,000개 이상  
- **구성**  
  - **hosp** : 병원 전반(EHR)
  - **icu** : ICU 임상 정보

---

## 데이터 전처리 & 주요 테이블

### 1. Admissions
- 각 환자의 입원 데이터  
  - **입원부터 퇴원까지의 시간**(`hrs`) 계산: \(`dischtime - admittime`\)  
  - **입원유형**, **입원위치**, **퇴원위치** 등 범주형 변수 → *One-hot encoding*  

### 2. Diagnoses-ICD
- 환자가 병원에 입원한 동안 청구된 진단 정보
- **d-icd-diagnoses** 테이블에서 `long-title`에 `acute kidney failure` 키워드가 포함된 경우 AKI로 간주 → AKI 라벨(1) 부여

### 3. HCPCSevents
- 청구 이벤트 관련 (다른 테이블과 중복되는 정보가 많을 것으로 예상 → 사용 X)

### 4. Procedures-ICD
- 입원 중 청구되는 진료 절차  
- AKI 환자가 받은 절차 총 10,389가지 vs 전체 14,911가지  
- 유의미한 정보로 활용하기 어렵다고 판단 → *사용 X*

### 5. Labevents
- 실험실 측정 결과(검사)  
1. AKI 환자 vs 비AKI 환자의 검사 코드 빈도 비교 → 모비율 차 검정 → *p-value < 0.05*인 코드 추출  
2. 추출된 검사들 중에서 진행 빈도가 평균 이상인 검사만 최종 선정  
3. 각 검사 결과를 **0(미실시)**, **1(정상)**, **2(비정상)**으로 라벨링하여 열 생성

### 6. DRGcodes
- 각 **입원 id**별 입원 이유(`drg-code`)와 심각도(`drg-severity`, `drg-mortality`) 정보
- Labevents와 동일한 방식으로 유의미한 drg-code 88개를 추출
- 각 입원 id에 대해 추출된 88가지 코드 중 해당되는 것에 **1**, 아니면 **0**을 할당  
- `drg-severity`, `drg-mortality` 열도 추가로 활용

### 7. EMAR (Prescriptions)
- 약물 투여 기록  
- AKI 진단 이력이 있는 환자의 prescriptions는 6,241가지, 전체는 6,588가지로 분산이 크고 정보 추출이 어려움 → *사용 X*

### 8. Microbiology
- 감염성 증식 및 항생제 감수성 테스트 정보  
- `test-itemid`, `org-itemid`, `ab-itemid`를 concat하여 6,478가지 검사 식별  
- labevents와 동일한 방식으로 유의미한 621가지 검사 추출  
- 각 입원 id마다 검사 결과를 0(미검사) / 1(민감) / 2(중간) / 3(내성)으로 정리

### 9 ~ 13. 기타 EHR 테이블
- 예) Chartevents(생체 신호), Input/Output events(약물·수액 투여, 배설물 등)
- 동일한 방식으로 **유의한 코드만 추출**하고, 각각의 입원 id에 대한 결과 열 생성

## 모델 적용 시 문제점

- **차원 문제**: 범주형 변수를 다항(one-hot)으로 인코딩 시 수천 개 열이 생성 → **Sparse**하며 **차원 너무 큼**  
- **데이터 불균형**: 8:2로 극단적이진 않지만, 실제 **이상치**로 간주하기엔 미묘한 분포  
- **시계열 특성 반영 어려움**: 과거 진단이나 치료가 미래 진단에 영향을 주는데, 이를 모델이 직접 고려하기 어려움  
- **Censoring 이슈**: 관찰이 끝나기 전에 진단이 되지 않은 경우(검사 미실시) 등으로 인한 정보 손실  

---

## 결론 및 정리

- **DeepSAD**를 통해 AKI를 **이상치(Anomaly)** 로 정의하고 예측을 시도하였으나,  
  - **고차원 Sparse** 데이터,  
  - **시계열 특성 미반영**,  
  - **Censoring** 등으로 인해 기대만큼의 성능이 나오지 않음  
- 이에 더해 **공정성(Fairness) 로스**를 결합함으로써 **특정 집단**이 예측 성능에서 불이익을 받지 않도록 하는 접근 방안을 추가 모색
